Creación de una API RESTful

1. Objetivo

Diseñar e implementar una API RESTful que orqueste un ciclo completo de vida para recursos de negocio, habilitando operaciones CRUD (Crear, Leer, Actualizar y Eliminar) y garantizando consistencia transaccional. El entregable debe evidenciar dominio de patrones arquitectonicos modernos y buenas practicas de ingenieria de software.

Stack tecnologico de implementacion

La tecnologia protagonista es NestJS, framework modular construido sobre el runtime de Node.js (modelo event-driven y non-blocking I/O documentado en nodejs.org) y que aprovecha Express como adaptador HTTP (referencia oficial en expressjs.com) para habilitar middlewares de validacion, serializacion y manejo centralizado de errores.

  • Runtime: Node.js LTS 20.x con gestor de paquetes npm, habilitando scripts de compilacion, linting y ejecucion en caliente (npm run start:dev).
  • Framework HTTP: Express 4.x integrado con middleware express.json() para parser de solicitudes REST y gestion de rutas modulares.
  • Lenguaje: TypeScript 5.7 con reglas estrictas de tipado y soporte para decoradores en capas de infraestructura.
  • Documentacion: Swagger UI expuesto en /api, generando especificaciones OpenAPI a partir de decoradores y anotaciones de la capa de controlador.

2. Requisitos Técnicos

Tecnologías Requeridas

  • Framework: Express.js (Node.js)
  • Base de Datos: MySQL
  • Herramienta de Pruebas: Postman
  • Lenguaje: JavaScript/TypeScript

Dependencias Clave

  • @nestjs/core, @nestjs/common: kernel del framework NestJS utilizado para orquestar modulos, controladores y servicios.
  • mysql2: cliente SQL con soporte para conexiones pool y promesas.
  • @nestjs/swagger y swagger-ui-express: generacion de especificaciones OpenAPI y renderizado de la UI documental.
  • class-validator/class-transformer (opcional): validacion declarativa de DTOs y transformacion de payloads.

Configuracion de compilacion

  • TypeScript: Target ES2020, decoradores habilitados, strict mode activo.
  • Scripts npm: npm run start:dev (modo watch), npm run build (transpilacion), npm run test (Jest).
  • Linting/Formato: ESLint 9 + Configuracion Prettier para mantener convenciones de estilo.

Infraestructura de ejecucion

  • Runtime: Node.js LTS 20.x
  • Gestor de paquetes: npm 10.x
  • Servidor HTTP: Express como adaptador interno de NestJS
  • Documentacion en vivo: Swagger UI disponible en /api

3. Especificación de Endpoints

Definir rutas para cada operación CRUD con base en el módulo de medicamentos implementado en NestJS. La API expone 7 endpoints principales que cubren todo el ciclo de vida del recurso:

Operaciones CRUD Básicas

GET /

Exponer un mensaje de bienvenida (hola mundo) para verificar disponibilidad del servicio

GET /medicamentos

Listar todos los medicamentos disponibles

GET /medicamentos/{id}

Consultar el detalle de un medicamento por identificador

POST /medicamentos

Registrar un nuevo medicamento en el inventario

PUT /medicamentos/{id}

Actualizar la informacion completa de un medicamento existente

DELETE /medicamentos/{id}

Eliminar lógicamente un medicamento

GET /medicamentos/estado/{estado}

Filtrar medicamentos por estado operativo (activo, inactivo, descontinuado)

Tabla de Especificación de Endpoints

Método Endpoint Descripción Parámetros
GET / Mostrar mensaje de saludo para comprobar estado de la API Query: ninguno
GET /medicamentos Listar todos los medicamentos Query: ninguno
GET /medicamentos/{id} Obtener medicamento por identificador Path: id
POST /medicamentos Crear nuevo medicamento Body: JSON con campos de medicamento
PUT /medicamentos/{id} Actualizar informacion existente Path: id, Body: JSON

GET /medicamentos/estado/{estado} Filtrar por estado del medicamento Path: estado
